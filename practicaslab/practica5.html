Práctica: Node MCU ESP32 – Sensores Digitales, Analógicos e Inteligentes

Resumen

En esta práctica, buscamos conocer y comprobar el funcionamiento de diversos sensores conectados al módulo NodeMCU ESP32. Los sensores utilizados incluyen un botón (sensor digital), un potenciómetro (sensor analógico) y un DHT22 (sensor inteligente). Además, configuraremos el entorno de VOKWI para realizar simulaciones y programaremos el NodeMCU ESP32 utilizando el IDE de Arduino para validar el funcionamiento de los sensores en un entorno real.

Introducción

La tecnología de sensores es fundamental en la construcción de proyectos de electrónica y automatización. En esta práctica, nos enfocaremos en el uso del módulo NodeMCU ESP32 para interactuar con diferentes tipos de sensores. Configuraremos un entorno de simulación en WOKWI y realizaremos pruebas con sensores digitales, analógicos e inteligentes. Luego, trasladaremos estos conocimientos a un entorno real utilizando el IDE de Arduino para programar y verificar el funcionamiento de los sensores con un NodeMCU ESP32 real.

Materiales

  • Node MCU ESP32
  • LED, resistencia de 220 Ohms
  • Botón y resistencia de 1K Ohm
  • Sensores: Potenciómetro, DHT22, PIR
  • Protoboard y cables
  • Wokwi
  • Arduino IDE

Desarrollo

Objetivo General

Conocer, identificar y comprobar el funcionamiento de los sensores digitales, analógicos e inteligentes en el módulo NodeMCU ESP32.

Objetivos específicos

  • Utilizando la información e instrucciones de la presentación, configurar el entorno de VOKWI y probar los 3 ejemplos de sensores: Sensor digital (botón), Sensor Analógico (Potenciómetro) y Sensor Inteligente (DHT11).
  • Desarrollar un programa de monitoreo de un sensor digital PIR, basándote en el ejemplo y la documentación del sensor: https://docs.wokwi.com/parts/wokwi-pir-motion-sensor.
  • Desarrollar un programa de monitoreo de un sensor analógico “Slide Potentiometer”, basándote en el ejemplo y la documentación del sensor: https://docs.wokwi.com/parts/wokwi-slide-potentiometer.
  • Desarrollar un programa con un nuevo sensor inteligente que utilice una librería externa.
  • Utilizando la información de la siguiente liga, configurar el IDE de Arduino para poder programar el NodeMCU directamente desde el entorno de Arduino, instalando las herramientas necesarias. Realiza los mismos objetivos específicos del simulador, utilizando un NodeMCU ESP32 real. https://randomnerdtutorials.com/getting-started-with-esp32/

Resultados

Ejercicios en WOKWI

Sensores digitales

Iniciamos probando un botón, la conexión fue sencilla, utilizamos los pines 33 y 32 para conectar el botón y un led. Cada vez que presionamos el botón, el led se enciende.


Para el siguiente ejercicio usamos un sensor PIR. Este sensor es un dispositivo electrónico que detecta cambios en los niveles de radiación infrarroja emitida por objetos en su campo de visión, utilizado comúnmente en sistemas de detección de movimiento.


El siguiente sensor que utilizamos fue un sensor IR, este sensor es un dispositivo que emite y/o detecta radiación infrarroja para medir la temperatura de objetos o detectar obstáculos, variaciones de distancia y movimiento. El funcionamiento es un poco parecido al sensor PIR. De igual manera colocamos un led para detectar cuando el sensor esta funcionando.


Sensores analógicos

Para iniciar con los sensores analógicos, utilizamos un potenciometro normal, la conexión es sencilla ya que solo tienes 3 pines. Añadimos un led para probar su correcto funcionamiento, cuando el valor analógico es mayor a 2048 el led se encenderá.


Para el siguiente ejercicio utilizamos un potenciometro deslizante, el funcionamiento es el mismo que el de uno normal. Utilizamos el mismo código del anterior ejercicio, solo cambiamos el sensor.


El último sensor analógico que utilizamos fue un sensor LDR. Este sensor es un dispositivo pasivo cuya resistencia varía inversamente con la cantidad de luz incidente sobre su superficie.


Sensores inteligentes

Como sensor inteligente utilizamos el sensor DHT22 que mide temperatura y humedad del ambiente, de igual manera, colocamos un led y este prende cuando el valor de la humedad es mayor a 50.

Conclusiones

A lo largo de esta práctica, logramos familiarizarnos con el uso del NodeMCU ESP32 y la integración de diversos sensores. La simulación en VOKWI nos permitió entender el comportamiento de los sensores antes de implementarlos en un entorno real. Posteriormente, al configurar el IDE de Arduino y programar el NodeMCU ESP32, pudimos validar la funcionalidad de los sensores y su aplicación en proyectos de electrónica. Esta experiencia nos ha proporcionado una base sólida para futuros proyectos de automatización y control utilizando el NodeMCU ESP32.
